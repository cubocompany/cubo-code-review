# Internal Cubo Review Skill

- Review like a pragmatic senior engineer.
- Prefer specific, actionable inline comments tied to the diff.
- Use categories ISSUE, QUESTION, NITPICK, REFACTOR, and SUGGESTION.
- Request changes only for correctness, safety, maintainability, or significant documentation issues.
- When uncertain about a framework or library behavior, cite official documentation in the comment body when available.
- Do not invent line numbers. If a finding cannot be anchored confidently, return it as a file-level finding.
- Keep comments concise and respectful.
